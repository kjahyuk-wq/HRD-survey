# 지방공공기관 신규자 과정 (type: 'newcomer') 설계·구현

> 작성: 2026-07-21 · 상태: **구현 완료 (배포 필요)**
> 근거: 신규자(2기) 종이 설문지(surveyPaper2.pdf) + 담당자 확인 사항.

## 1. 개요

지방공공기관 신규자 과정은 설문 문항이 표준 설문과 완전히 달라 별도 과정 유형으로 분리.
- 과정 문서: `courses/{id}.type = 'newcomer'`
- 회차(rounds) 없음 — 단기과정처럼 기수마다 과정 1개 생성, 학생당 1회 응답(`completed`)
- **반(분반)**: 일부 과목은 교육생이 둘로 나뉘어 강사 2명이 진행
  → 관리자가 **강사·수강생 명단에 반을 지정**, 학생은 공통 강사 + 본인 반 강사만 평가
  (중견리더의 학생 자기선택(electives) 모델과 다름 — 반 수가 적어 관리자 지정이 담당자 결정)

## 2. 담당자 확인 사항 (2026-07-21)

| 항목 | 결정 |
|---|---|
| 종이 설문의 과목별 '시간배정(과다/적당/부족)' | **웹에는 넣지 않음** — 만족도 5점만 |
| 반 지정 방식 | **관리자가 명단(엑셀 C열)에 지정** (학생 자기선택 아님) |
| 반별 통계 | **과정 만족도(고정 문항)도 반별로 각각** 나와야 함 → 응답에 `groupName` 저장, 통계 반 필터 |
| 6-1 / 7-1 (불만족 시 개선사항 주관식) | 항상 표시, 선택 입력 |

## 3. 문항 (종이 설문지 기준, 웹 저장 키)

객관식 16문항 — 척도 12개(웹 1=매우불만족~5=매우만족), 선택형 4개:

| 키 | 종류 | 내용 |
|---|---|---|
| nq1 | 척도 | 교육기간 |
| nq2 | 선택 | 교육기간 유지 의견 (찬성/보통/반대) |
| nq3 | 선택 | 입교 동기 (본인 희망/교육 담당부서의 배정(권유)) |
| nq4 | 척도 | 가치관 확립 도움 (그렇다 척도) |
| nq5~nq14 | 척도 | 교과편성/소양교육/직무교육/강사선정/교재/운영/교육효과/식당/교육시설/편의시설 |
| nq15, nq16 | 선택 | 연령 / 성별 |

주관식: `nq6_comment`(6-1), `nq7_comment`(7-1), `comment1~3`(소감/개선/건의 — 표준과 동일).
문항 정의 단일 소스: `admin-utils.js`의 `NC_SURVEY` / `getSurveyConfig('newcomer')`.
Cloud Function 검증 키 목록(`NC_SCALE_KEYS`/`NC_CHOICE_KEYS`)과 **반드시 1:1 유지**.

## 4. 데이터 모델

- 강사: `courses/{id}/instructors/{iid}.group: string` (선택) — 비움 = 공통(전원 평가), 값 = 그 반 학생만 평가.
  분반 과목은 같은 강의명으로 강사 2명 등록하고 각각 반을 지정 (예: 노무관리|A강사|1반, 노무관리|B강사|2반).
- 수강생: `courses/{id}/students/{sid}.group: string` (선택) — 관리자 지정. **미지정 학생은 공통 강사만 평가**하므로 반드시 지정할 것.
- 응답: `courses/{id}/responses/{rid}` — `nq1~nq16, nq6_comment, nq7_comment, instructors{}, comment1~3, groupName`(서버가 학생 doc 에서 박음 — 클라이언트 조작 불가).

## 5. 관리자 흐름

- 과정 추가 시 유형 '지방공공기관 신규자' 선택.
- 강사 엑셀: A열 강의명 / B열 강사명 / **C열 반(비우면 공통)**. 행 순서 = 설문·export 순서.
- 수강생 엑셀: A열 교번 / B열 이름 / **C열 반**. 수강생 표에서 반 인라인 수정 가능.
- 통계: 반이 있으면 반 필터 셀렉트 노출. 반 선택 시 응답·학생·강사(공통+그 반)가 모두 좁혀져
  **고정 문항 만족도도 반별로** 나옴. 엑셀 export 도 필터 상태 그대로.
- 미리보기: 신규자 문항 세트 + 반 필터.

## 6. 엑셀 export (신규자)

- 업로드용 '공공기관신규자' 시트 — **"공공기관 신규자 설문분석 결과폼" 서식과 동일** (2026-07-22 실물 대조 확인):
  - 순번 + 문항 1~16 + **빈칸 4열(17~20, 주관식 자리)** + 21번부터 강사 (강사 수만큼 열 증가)
  - **모든 값은 순수 숫자**: 척도는 6-v 반전(서식 ①=매우만족), 선택형은 보기 번호, 강사도 6-v 숫자
  - 강사는 등록 순서 고정, 0응답도 빈 열 유지
- 주관식 시트: 순번 / 6-1 / 7-1 / 소감 / 개선 / 건의.
- 표준(단기/중견리더) 과정의 기존 `n_0` 텍스트 서식은 그대로 유지 (신규자만 분기).

## 7. 영향 파일

`admin-utils.js`(문항 정의·config), `index.html`+`main.js`(학생 흐름),
`attendance/functions/index.js`(submitSurveyResponse 검증·groupName),
`admin.html`, `admin-courses.js`(유형·강사 반), `admin-students.js`(수강생 반),
`admin-stats.js`(config 기반 통계 + 반 필터), `admin-excel.js`(export 분기), `admin-preview.js`.
firestore.rules 변경 없음 (반 필드는 admin 만 쓰기).

## 8. 배포

1. 호스팅: 루트에서 `firebase deploy --only hosting`
2. **Cloud Function 필수**: `attendance/` 에서 `firebase deploy --only functions:submitSurveyResponse`
   — 배포 전의 함수는 nq 문항을 거부하므로 신규자 설문 제출이 실패함.
