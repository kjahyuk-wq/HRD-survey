# 중견리더양성과정 만족도 — "공통/선택(카테고리)" 모델 개편 계획

> 작성: 2026-05-18 · 상태: **설계 확정 단계, 미구현** (오늘은 코드 안 고침)
> 이 문서는 git 으로 다른 컴퓨터에서도 열람 가능. 구현 재개 시 이 문서를 먼저 읽을 것.

## 1. 문제 (왜 개편하나)

지난 작업에서 "조(분반)마다 강사가 다르다"는 요구를 **회차 분반(`groups`) 모델**로 구현함:
학생마다 고정 `group` 을 박아두고 그 분반 강사만 필터링.

실제 운영은 "고정 팀(조)"가 아니라 **각 교육생이 선호에 따라 선택과목을 골라 듣는 구조**라
모델이 안 맞음 → 담당자가 조별 강사 명단 엑셀을 **20~30개** 따로 업로드해야 함 (가장 큰 고통).

### 실제 운영 데이터 (담당자 설명, 2026-05-18)
- 중견리더양성과정 전체 교육생 ≈ **76명**, 만족도 조사 **2개월에 1회**
- 만족도 대상 강사 ≈ **72명**
  - **공통 강사 ≈ 53명**: 전 교육생이 무조건 평가
  - **선택 강사 ≈ 19명**: 본인이 들은 것만 평가, 카테고리로 묶임
    - 체련활동: 골프 / 피지컬 트레이닝 / 탁구
    - 취미활동: 요리 / 사진 / 통기타 / 캘리그라피 / 미술
    - 외국어: 영어 / 일어
    - 자격증: 심리상담사 / 바리스타 / 한국사
- 결과 업로드 엑셀은 **인재개발원 교육시스템의 강사 순서와 정확히 일치**해야 함
  (교육시스템이 a,b,c,d,e 순이면 우리 export 도 a,b,c,d,e 순)

## 2. 새 모델: 공통 / 선택(카테고리)

### 관리자 — 회차당 강사 엑셀 **1개**
| A열 강의명 | B열 강사명 | C열 구분 |
|---|---|---|
| 리더십개론 | 홍길동 | (비움) 또는 `공통` |
| … 공통 강사 … | | `공통` |
| 골프 | 김프로 | `체련활동` |
| 탁구 | 이프로 | `체련활동` |
| 요리 | 박셰프 | `취미활동` |
| 영어 | Smith | `외국어` |
| 한국사 | 정선생 | `자격증` |

- C열 비움/`공통` → 전원 평가
- C열 = 카테고리명 → 선택 강사
- **엑셀 행 순서 = 설문 표시 순서 = 결과 export 순서**
  → 인재개발원 서식 순서대로 엑셀만 만들면 자동 정합
- 기존 분반(조) 정의 UI / 체크박스 → 카테고리 모델로 교체

### 학생 — 설문 흐름
로그인 → (중견리더면) 회차 선택 → **[신설] "수강한 선택활동 선택" 화면**
→ 공통 강사 전원 + 본인 선택분만 설문 노출.

선택 화면은 **카테고리별 섹션 + 체크박스**(평면 19개 나열 금지):
```
체련활동   ☐ 골프  ☐ 피지컬 트레이닝  ☐ 탁구
취미활동   ☐ 요리  ☐ 사진  ☐ 통기타  ☐ 캘리그라피  ☐ 미술
외국어     ☐ 영어  ☐ 일어
자격증     ☐ 심리상담사  ☐ 바리스타  ☐ 한국사
```

### 선택 영속화 (중요)
교육생이 **처음 한 번 선택하면 학생 문서에 저장** → 올해 11월(과정 종료)까지 유지.
이후 회차는 선택 화면 건너뛰고 저장된 선택분 자동 적용. (담당자/필요 시 변경만 가능)
저장 위치 후보: `courses/{id}/students/{sid}.electives: string[]`
(선택 강사 키 배열 — `education__name` 또는 강의명 기준).

## 3. 확정된 결정

| 항목 | 결정 |
|---|---|
| 조(분반) 모델 | **공통/선택(카테고리) 모델로 완전 교체** |
| 카테고리 내 중복 수강 | **제한 없이 여러 개 선택 허용** (체크박스) |
| 결과 export 강사 열 | **공통+선택 전원 고정 순서, 응답 0명도 빈칸 자리 유지** |
| 선택 영속 | 학생 문서에 저장, 11월까지 재선택 없이 유지 |
| 선택 화면 | 카테고리별 섹션 UI |

## 4. 담당자에게 재확인 필요 (불확실 → 안전값으로 가정)
- 카테고리당 1개만인지 여러 개인지 → **여러 개 허용**으로 가정 (제약 없음이 안전)
- 인재개발원 업로드 서식이 강사 고정 열인지 → **고정 열**로 가정
  (선택 강사가 응답 0명이어도 그 자리 비워서 출력)

## 5. 영향 받는 파일 (구현 시)
- `admin-rounds.js` — 분반 CRUD/체크박스 → 카테고리 모델. 강사 엑셀 C열=구분 파싱
  (`uploadRoundExcelInstructors`, `parseRoundInstExcelFile`, `renderRoundInstructorsPanel`)
- `admin-students.js` — 분반 셀렉트/엑셀 C열(분반) 관련 정리, 학생 문서 electives 표시·수정(선택)
- `main.js` — `selectRound` 분반 필터 → 카테고리 필터, **선택활동 화면 신설**,
  학생 문서 electives 읽기/저장, 미선택 시 1회 노출 후 영속
- `admin-stats.js` — `loadStats` 의 `groups` 필터 → 카테고리 기반, `computeStats`
  의 `orderedInstructorKeys` 가 전체 강사(0응답 포함) 고정 순서 보장
- `admin-excel.js` — `exportStatsExcel`/`exportResultsExcel` 가 전체 강사 고정 순서
  (응답 0명 강사도 열 유지), 분반 라벨 → 제거 또는 무의미화
- `admin-preview.js` — 미리보기 강사 목록 카테고리 반영
- `admin.html` — 회차 강사 패널/통계 탭 UI 문구·요소 교체
- `firestore.rules` — 학생 문서 `electives` 필드 허용, `response.groupName` 정리
- `attendance/functions/index.js` — `submitSurveyResponse` 는 강사 키 일반 검증만 하므로
  거의 무변경. 단 학생 문서 `electives` 갱신을 서버 트랜잭션에 넣을지 검토
- 기존 운영 분반 데이터 마이그레이션/정리 필요 (leadership 2026-05-10 운영 진입)

## 6. 마이그레이션 메모
- 기존 강사 `groups: []` / `groups:['common']` → "공통"으로 해석 (호환 처리)
- 기존 회차 `groups: string[]` (분반 정의) → 제거 또는 무시
- 기존 응답 `response.groupName` → 통계 영향 없게 무시 처리

## 7. 재개 시 첫 단계
1. 담당자에게 4절 2개 항목 최종 확인
2. 데이터 모델(카테고리/electives 필드명) 확정
3. 관리자 엑셀 C열 파싱 → 학생 선택 화면 → export 순서 순으로 구현
4. 운영 분반 데이터 마이그레이션 스크립트
